(inner) localLanguageName('language', 'wordCount') → {string}

localLanguageName - Computes the used language of the editor. In order to compute it the function will listen for every change of the variables language and wordCount. Therefore with each new word the language will be evaluated. It may happens that a language is not detected. In this case the return value will be empty. Otherwise the local language will be returned e.g. German will be returned as 'Deutsch'.
Parameters:
Name Type Description
'language' string The language of the content as calculated before
'wordCount' number the count of words within the content. The detection does not work properly with short sentences. Therefore it might change in future that you need to provide a defined number of words before the language will be derived.
Source:
  • pods/components/variable-editor/component.js, line 39
Returns:
The calculated language as local string. In case the algorithm could not determine a language an empty string will be returned
